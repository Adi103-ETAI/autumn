    11Ôćĺ## 0. Architecture overview
    12Ôćĺ
    13Ôćĺ- A single `function registerIpc()` at `index.js:18617` wires **all** `ipcMain.handle`/`ipcMain.on` listeners (226 registrations total). There is no per-subsystem registration elsewhere ÔÇö every channel is listed inside this one function.
    14Ôćĺ- Secret material is held in two separate stores:
    15Ôćĺ  1. **`settings.json`** in `userData` ÔÇö JSON, key/value secrets are encrypted with Electron `safeStorage` and the plaintext copy is also kept in-process in module-level `inMemory*` variables.
    16Ôćĺ  2. **`supabase-session.json`** in `userData` ÔÇö the Supabase auth session (JWT refresh token etc.), encrypted via a `fileStorage` adapter backed by `safeStorage`.
    17Ôćĺ- `safeSend(channel, payload)` (`index.js:435`) fans a message out to all open BrowserWindows via `win.webContents.send`. Auth-change and analytics events use it.
    18Ôćĺ- Validation helper `assertUuid(id)` (`index.js:18613`) enforces `/^[0-9a-f-]{8}-ÔÇŽ-ÔÇŽ-ÔÇŽ-ÔÇŽ$/` (canonical UUID) and throws `"Invalid canvas id."` for anything else. `CANVAS_ROOM_RE` (`index.js:18612`) splits liveblocks room names `canvas-<uuid>`.
    19Ôćĺ- `maskSecrets(s)` (`index.js:18647`) returns a copy of the settings object with every secret replaced by `"ÔÇóÔÇóÔÇóÔÇó"` if present (else `undefined`). Used by `settings:get` and `settings:set` so the renderer never receives plaintext secrets.

    90Ôćĺ**Sharing model:** a canvas is "shared" by setting `is_public=true` on its `desktop_canvases` row.    340Ôćĺ## 2. AUTHENTICATION
   341Ôćĺ
   342Ôćĺ### 2.1 Three distinct auth paths
   343Ôćĺ
   344Ôćĺ1. **Supabase email/password** (`auth:sign-in`/`auth:sign-up`) ÔÇö wrapped by `authSignIn`/`authSignUp` (`index.js:5862`, `5869`) calling `client().auth.signInWithPassword` / `signUp`.
   345Ôćĺ2. **GitHub OAuth** (`auth:sign-in-github`) ÔÇö `authSignInWithGitHub` (`index.js:5878`).
   346Ôćĺ3. **Bring-your-own API key / October Cloud token** ÔÇö *not* IPC-guest auth; these are stored in **Settings** (`providerMode`, keys) and used directly by the agent harnesses. The `auth:*` Supabase flow is independent of `providerMode`.
   347Ôćĺ
   348ÔćĺSeparately, **Claude.ai OAuth** (`oauth:claude-login`) writes native Claude Code credentials (`index.js:5723`).
   349Ôćĺ
   350Ôćĺ### 2.2 Supabase client & session storage
   351Ôćĺ
   352Ôćĺ- `SUPABASE_URL = "https://latwxiqjgvluiddckvmj.supabase.co"` (`index.js:5797`)
   353Ôćĺ- `SUPABASE_ANON = "eyJÔÇŽ(JWT anon key)ÔÇŽ"` (`index.js:5798`) ÔÇö a **public anon key** (RLS-protected, not secret).
   354Ôćĺ- Client created lazily in `client()` (`index.js:5849`) with `auth: { persistSession:true, autoRefreshToken:true, detectSessionInUrl:false, storage: fileStorage, flowType:"pkce" }`.
   355Ôćĺ- Session persisted via `fileStorage` adapter (`index.js:5816`):
   356Ôćĺ  - `getItem(key)` / `setItem(key,value)` / `removeItem(key)` encrypted with `electron.safeStorage.encryptString`/`decryptString`.
   357Ôćĺ  - Encrypted blob written to `userData/supabase-session.json` (mode `384` = `0600`, written atomically via tmp+rename in `writeEnc`, `index.js:5810`).
   358Ôćĺ  - **Fallback when `safeStorage.isEncryptionAvailable()` is false:** an in-memory `memStore` object only (no disk persistence).
   359Ôćĺ- Current Supabase JWT is read by `authAccessToken()` (`index.js:5957`) Ôćĺ used to authenticate October-proxied calls (liveblocks, deepgram).
   360Ôćĺ- `toUser(u)` (`index.js:5859`) maps a Supabase user to `{id, email}`.
   361Ôćĺ- `onAuthStateChange(cb)` (`index.js:6188`) registers the listener that drives `event:auth-changed`.
   362Ôćĺ
   363Ôćĺ### 2.3 GitHub OAuth flow (`authSignInWithGitHub`, `index.js:5878`)
   364Ôćĺ
   365Ôćĺ- PKCE: `signInWithOAuth({provider:"github", options:{redirectTo: GH_REDIRECT, skipBrowserRedirect:true}})` returns an authorize URL; main opens it in the user's browser via `electron.shell.openExternal`.
   366Ôćĺ- A local HTTP server listens on **`127.0.0.1:54547`** (and `::1`) ÔÇö `GH_PORT = 54547`, `GH_REDIRECT = "http://localhost:54547/callback"`, `GH_TIMEOUT_MS = 300000` (`index.js:5874-5876`).
   367Ôćĺ- Browser redirects to `/callback?code=ÔÇŽ`; main calls `client().auth.exchangeCodeForSession(code)` (PKCE). On success returns the user. Two servers are bound (IPv4 + IPv6) for resilience; the second silently no-ops on bind error.
   368Ôćĺ
   369Ôćĺ### 2.4 Claude.ai OAuth (`claudeOAuthLogin`, `index.js:5723`)
   370Ôćĺ
   371Ôćĺ- Constants (`index.js:5659-5665`):
   372Ôćĺ  - `CLIENT_ID = "9d1c250a-e61b-44d9-88ed-5944d1962f5e"`
   373Ôćĺ  - `AUTHORIZE_URL = "https://claude.ai/oauth/authorize"`
   374Ôćĺ  - `TOKEN_URL = "https://console.anthropic.com/v1/oauth/token"`
   375Ôćĺ  - `PORT = 54545`, `REDIRECT_URI = "http://localhost:54545/callback"`
   376Ôćĺ  - `SCOPES = "org:create_api_key user:profile user:inference"`
   377Ôćĺ- Generates PKCE `verifier`/`challenge` + `state`, opens `AUTHORIZE_URL` in browser, listens on `54545/callback`, exchanges code at `TOKEN_URL` (`exchangeCode`, `index.js:5696`), then writes `~/.claude/.credentials.json` via `writeClaudeCredentials` (`index.js:5667`) under `claudeAiOauth: {accessToken, refreshToken, expiresAt, scopes, subscriptionType:"unknown"}`. File mode `0600`.
   378Ôćĺ
   379Ôćĺ### 2.5 `providerMode` model (October Cloud token + BYO keys)
   380Ôćĺ
   381Ôćĺ`providerMode` (default `"claude-code"`) selects how the agent harness authenticates to the model API. `buildEnv(settings)` (`index.js:2471`) shows the contract:
   382Ôćĺ- `"api-key"` Ôćĺ `env.ANTHROPIC_API_KEY = settings.apiKey` (the user's own Anthropic key).
   383Ôćĺ- `"october-cloud"` Ôćĺ `env.ANTHROPIC_BASE_URL = settings.octoberBaseUrl || "https://api.october.dev/v1"` and `env.ANTHROPIC_AUTH_TOKEN = settings.octoberToken` (October proxies Anthropic; the October Cloud token is the bearer).
   384Ôćĺ- `"claude-code"` (default) Ôćĺ uses the locally-installed Claude CLI credentials (`~/.claude`), no env vars set.
   385Ôćĺ
   386Ôćĺ`getAuthStatus()` (`index.js:3410`) inspects the filesystem for each harness's native credentials (Claude, Codex, Hermes, Cursor, Grok, OpenCode, Gemini, Cline, Qwen, Goose, Copilot, Campfire) and reports `installed`/`loggedIn`/`hasKey` flags plus `keychainAvailable = safeStorage.isEncryptionAvailable()`.
   387Ôćĺ
   388Ôćĺ### 2.6 Session state tracking
   389Ôćĺ
   390Ôćĺ- Supabase session = the canonical "signed in to October" state, persisted in `supabase-session.json`. `authStatus()` returns the current user.
   391Ôćĺ- `event:auth-changed` is emitted on every Supabase auth-state change; the renderer uses it to update UI, and main uses it to tear down `codeSync`/`busCloudSync` on sign-out.
   392Ôćĺ- October Cloud / BYO keys are *settings*, not "sessions" ÔÇö they have no refresh/expire lifecycle in the app (only the harness rejects them at runtime, surfaced by `describeAgentError`, `index.js:2451`).

   394Ôćĺ### 2.7 `maskSecrets` (definition `index.js:18647`)

   396Ôćĺ```js

   417Ôćĺ### 2.8 Secret encryption at rest
   418Ôćĺ
   419Ôćĺ- `encryptKey(key)` (`index.js:3386`): if `key === ""` returns `{stored:undefined, memory:undefined}`; else if `safeStorage.isEncryptionAvailable()` returns `{stored: safeStorage.encryptString(key).toString("base64"), memory: key}`; else `{stored:undefined, memory:key}` (plaintext only in memory, never written).
   420Ôćĺ- `decryptKey(enc, mem)` reverses it: if `enc` present it decrypts; otherwise returns the in-memory copy.
   421Ôćĺ- In-memory copies live in module vars declared at `index.js:3198-3207` (`inMemoryApiKey`, `inMemoryOpenaiApiKey`, `inMemoryCursorApiKey`, `inMemoryGrokApiKey`, `inMemoryGeminiApiKey`, `inMemoryOctoberToken`, `inMemoryPostBridgeApiKey`, `inMemoryShopifyThemeToken`, `inMemoryManusApiKey`, `inMemoryFigmaToken`, and continues for `inMemoryShortcutToken`, `inMemoryCanvaAccessToken`). Decryption happens only when the renderer/agent actually needs the plaintext.
   422Ôćĺ- `migrateLegacyOctoberToken` (`index.js:3244`) upgrades old `octoberToken` plaintext field Ôćĺ `octoberTokenEnc` on first read.

   537Ôćĺ
   538Ôćĺ- `track(event, props)` (`index.js:4748`):

   608Ôćĺ## 7. CONTROL-FLOW SUMMARY
   609Ôćĺ
   610Ôćĺ1. On boot, `registerIpc()` (`18617`) wires every listener. `onAuthStateChange` subscribes and emits `event:auth-changed`.
   611Ôćĺ2. Renderer calls `settings:get` Ôćĺ main returns **masked** settings; `settings:set` merges + re-encrypts secrets + masks on return.
   612Ôćĺ3. To sign in: renderer Ôćĺ `auth:sign-in` / `auth:sign-up` / `auth:sign-in-github` / `oauth:claude-login`. Supabase sessions persist in `supabase-session.json` (encrypted); Claude creds in `~/.claude/.credentials.json`.
   613Ôćĺ4. Agent runs read `providerMode`: `"api-key"` uses `apiKey` as `ANTHROPIC_API_KEY`; `"october-cloud"` uses `octoberToken` as `ANTHROPIC_AUTH_TOKEN` against `api.october.dev/v1`; `"claude-code"` uses local CLI creds.
   614Ôćĺ5. Cloud canvases: CRUD against Supabase `desktop_canvases`; sharing toggles `is_public`; share link is `october://canvas/<uuid>` resolved by october.dev. Code-sync bundles git repos into the `code-sync` storage bucket.
   615Ôćĺ6. Liveblocks/deepgram tokens: main POSTs to october.dev with the Supabase JWT (`postWithOctoberJwt`).
   616Ôćĺ7. Analytics: `track()` buffers until `analytics:ready`, then `event:analytics` Ôćĺ renderer Ôćĺ Segment.
