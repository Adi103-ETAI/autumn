   117Ôćĺ## 3. electron-updater / Auto-Update Mechanism
   118Ôćĺ
   119Ôćĺ### 3.1 Configuration ÔÇö `resources/app-update.yml`
   120Ôćĺ
   121Ôćĺ```
   122Ôćĺowner: harshsaver
   123Ôćĺrepo: october-desktop-releases
   124Ôćĺprovider: github
   125ÔćĺupdaterCacheDirName: october-desktop-updater
   126Ôćĺ```
   127Ôćĺ
   128ÔćĺThis is the **only** electron-updater config shipped. There is **no** `autoUpdater.setFeedURL(...)`, `allowPrerelease`, `channel`, or `getUpdateConfigPath` call in `out/main/index.js` ÔÇö the updater reads this file automatically (electron-updater looks for `app-update.yml` next to the asar at runtime).
   129Ôćĺ
   130Ôćĺ- **provider:** `github` Ôćĺ electron-updater queries the GitHub Releases API for the `latest.yml` / per-version `.yml` metadata and downloads the matching asset.
   131Ôćĺ- **update server URL (derived):** `https://github.com/harshsaver/october-desktop-releases/releases` and the Releases API `https://api.github.com/repos/harshsaver/october-desktop-releases/releases`.
   132Ôćĺ- **updaterCacheDirName:** `october-desktop-updater` Ôćĺ the downloaded installer is cached under `%LOCALAPPDATA%/october-desktop-updater/` (Windows) instead of the default `electron-updater` cache dir.
   133Ôćĺ- **asset naming convention (electron-builder github):** expects a `October-<version>-x64.exe` (or `-x64.nsis.7z`/`.exe`) plus a `latest.yml` published with each GitHub Release. The release tag is the version (e.g. `v1.0.30`).

   182Ôćĺ

   184Ôćĺ
   185Ôćĺ### 3.5 Update flow summary
   186Ôćĺ
   187Ôćĺ1. App launches Ôćĺ `initUpdater()` (called at `19705` inside `app.whenReady`).
   188Ôćĺ2. Fetches `desktop_release_policy` from Supabase; may set `forced`.
   189Ôćĺ3. `autoUpdater.checkForUpdates()` hits GitHub Releases for `harshsaver/october-desktop-releases`.
   190Ôćĺ4. If a newer version exists Ôćĺ renderer shows prompt (`update:state` = `available`). User clicks "Update" Ôćĺ `update:start` Ôćĺ `downloadUpdate()`.
   191Ôćĺ5. `download-progress` streams percentage; on `update-downloaded` Ôćĺ `update:install` Ôćĺ `quitAndInstall(false, true)` runs the NSIS installer (the new `October-x64.exe` from the release) and relaunches.
   192Ôćĺ6. `autoInstallOnAppQuit=true` means even if the user ignores it, the download (if started) installs on quit.

   196Ôćĺ## 4. The NSIS Installer (`October-x64.exe`)
   197Ôćĺ
   198Ôćĺ### 4.1 Format & toolchain
   199Ôćĺ
   200Ôćĺ- **Nullsoft Install System v3.04** (Unicode), confirmed in the installer manifest: `description>Nullsoft Install System v3.04` and `SubType = NSIS-3 Unicode` (from the 7z listing of the exe).
   201Ôćĺ- Manifest `requestedExecutionLevel = asInvoker` (does **not** force admin at launch ÔÇö elevation is requested on demand via `UAC.dll`/elevation plugin).
   202Ôćĺ- The installer is a **self-extracting 7z archive wrapper**: the bulk of the payload is `app-64.7z` (315,488,971 bytes, `Type = 7z`, method `ARM64 LZMA2:20 LZMA:20 BCJ2`, solid, 453 files / 105 folders, expanded ~1.3 GB). 7z extracted it to `/tmp/october_work/app/` (the `October.exe` + `resources/` we analyzed).
   203Ôćĺ
   204Ôćĺ### 4.2 Plugins (from `$PLUGINSDIR`)
   205Ôćĺ
   206Ôćĺ| Plugin | Size | Role in this installer |
   207Ôćĺ|---|---|---|
   208Ôćĺ| `nsis7z.dll` | 434,176 | Decompresses the embedded `app-64.7z` payload into `$INSTDIR` (the Electron app tree). |
   209Ôćĺ| `UAC.dll` | 14,848 | User Account Control handling ÔÇö elevates specific install steps (e.g. writing to Program Files / creating machine-wide registry/shortcuts) without forcing the whole installer to run as admin. |
   210Ôćĺ| `nsDialogs.dll` | 9,728 | Builds custom Modern UI wizard pages (e.g. options / "installing..." progress). |

   216Ôćĺ
   217Ôćĺ### 4.3 Inferred install flow
   218Ôćĺ
   219Ôćĺ1. **NSIS stub runs** (asInvoker). Loads `UAC.dll`; if the chosen install path requires admin (default `Program Files`), it elevates the file-extraction/registration steps via UAC.

   239Ôćĺ
   240Ôćĺ### 5.1 elevate.exe (`resources/elevate.exe`, 107,520 bytes)
   241Ôćĺ
   242Ôćĺ- PE32 console executable, Intel i386. PDB path inside: `C:\Dev\elevate\bin\x86\Release\Elevate.pdb` Ôćĺ this is **Kai Liu's `elevate`** (a well-known ~100 KB open-source helper), **not** a Wega-built binary.
   243Ôćĺ- Purpose: re-launch a target process **elevated** (admin) using `ShellExecuteExW` with the `runas` verb (`requestedPrivileges` block present in its manifest). It does **not** do UAC prompting itself beyond delegating to Windows.

It is **distinct from** the installer's `UAC.dll` (which handles elevation *during installation*). On Windows it sits at `resources/elevate.exe` next to `app.asar`.

   247Ôćĺ

   297Ôćĺ## 7. How to Replicate the Build
   298Ôćĺ
   299ÔćĺTo produce an equivalent `October-x64.exe` + `app.asar` you need (in the **external build repo**, not shipped here):
   300Ôćĺ
   301Ôćĺ1. **Toolchain:** Electron `42.4.0` + `electron-builder` (NSIS target, x64) + `electron-updater@6.8.9`. TypeScript/React frontend bundled into `out/renderer` (Vite/etc.), main process compiled to `out/main/index.js`.
   302Ôćĺ2. **dependencies:** install exactly the 36 declared deps in ┬ž2 (plus their transitive native deps: `sharp`, `@img/*`, `onnxruntime-node`, `sherpa-onnx-*`, `@openai/codex*`, `node-pty`, `node-addon-api`).
   303Ôćĺ3. **asar + asarUnpack:** pack `out/` and `node_modules` into `app.asar`, but configure electron-builder to **unpack the native modules** (`node-pty`, `onnxruntime-node`, `sherpa-onnx-node`, `sherpa-onnx-win-x64`, `@img/sharp-win32-x64`, `@openai/codex-win32-x64`) into `app.asar.unpacked/node_modules/`. electron-builder does this automatically for any dep containing `.node`/`.dll`, which is why we see them duplicated in both locations.
   304Ôćĺ4. **app-update.yml:** place `owner: harshsaver / repo: october-desktop-releases / provider: github / updaterCacheDirName: october-desktop-updater` in `resources/`. Set `publish: [{ provider: "github" }]` in the electron-builder config so each release publishes `latest.yml` + `October-<version>-x64.exe` to the GitHub repo; electron-updater then auto-discovers them.
   305Ôćĺ5. **NSIS installer:** use electron-builder's `nsis` target with `oneClick:false` (or a custom `.nsi` using `nsis7z`, `UAC`, `nsDialogs`, `WinShell`, `StdUtils`, `System`, `nsExec` plugins). Embed the app as `app-64.7z`. Include `modern-wizard.bmp` as the MUI banner. Write the `october://` protocol registration (per-user `HKCU\Software\Classes\october`) in the install section. Generate `Uninstall October.exe`.
   306Ôćĺ6. **elevate.exe:** drop Kai Liu's `elevate.exe` into `resources/` so the running app can self-elevate for privileged actions.
   307Ôćĺ7. **Runtime policy:** populate the Supabase `desktop_release_policy` table (`platform`, `minimum_version`, `latest_version`, `message`) so `initUpdater()` can force mandatory upgrades.
