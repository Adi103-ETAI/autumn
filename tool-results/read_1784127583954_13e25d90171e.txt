    36→    36→Stage Summary:
    37→    37→- Repository fully understood. Ready to start the Autumn build.
    38→    38→- **Target product:** Autumn — an open-source spatial canvas app-builder (Electron + React 19 + Vite + Zustand + react-flow + Monaco + xterm + node-pty + Yjs + sherpa-onnx + Claude/Codex/ACP agents + own Supabase + AutumnBus HTTP+MCP).
    39→    39→- **Important architecture facts:**
    40→    40→  - Main process is the brain (~19.8k-line bundle in original).
    41→    41→  - Renderer is a thin-ish React shell over a 253-method `window.autumn` preload bridge (34 namespaces).
    42→    42→  - Agents = external CLI tools (Claude Code, Codex, OpenCode, Gemini, Cursor, Grok, Hermes, Pi, Campfire, Cline, Qwen, Goose, Copilot) launched as child processes with injected env; ACP-first dispatch with bespoke fallback.
    43→    43→  - Context bus (`AutumnBus`) = localhost HTTP server with `/mcp` (StreamableHTTP JSON-RPC) + `/hook/*` (session/stop/notify/pre-prompt/message-peer/task). Agents ride the bus via a zero-dep hook script.
    44→    44→  - Canvas: 7 node types (screen, chat, sticky, analytics, youtube, terminal, remotion), 2 edge types (bus, navigation).
    45→    45→  - "Commander" LLM prompt turns a user command into `DO_ACTIONS` plan (30 actions, verbatim recovered in `14-renderer-canvas.md`).
    46→    46→  - Substitution table: October Supabase → own Supabase; Liveblocks → Yjs; FluidAudio → sherpa-onnx; Deepgram → drop; PostHog → drop; `october://` → `autumn://`.
    47→    47→- **Sandbox constraint:** The actual development environment is the existing Next.js 16 project at `/home/z/my-project/`. Since Electron cannot run in this sandbox the way the original spec describes, the realistic deliverable here is a **Next.js web app** that demonstrates the Autumn concept — a spatial canvas + Commander + agent orchestration UI — using the open web stack we have (Next.js + Prisma + z-ai-web-dev-sdk for the LLM/intent layer). The deep RE notes remain the source of truth for behaviour and data model.
