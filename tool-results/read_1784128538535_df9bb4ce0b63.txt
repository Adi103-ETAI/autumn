    37Ôćĺ- Repository fully understood. Ready to start the Autumn build.
    38Ôćĺ- **Target product:** Autumn ÔÇö an open-source spatial canvas app-builder (Electron + React 19 + Vite + Zustand + react-flow + Monaco + xterm + node-pty + Yjs + sherpa-onnx + Claude/Codex/ACP agents + own Supabase + AutumnBus HTTP+MCP).
    39Ôćĺ- **Important architecture facts:**
    40Ôćĺ  - Main process is the brain (~19.8k-line bundle in original).
    41Ôćĺ  - Renderer is a thin-ish React shell over a 253-method `window.autumn` preload bridge (34 namespaces).
    42Ôćĺ  - Agents = external CLI tools (Claude Code, Codex, OpenCode, Gemini, Cursor, Grok, Hermes, Pi, Campfire, Cline, Qwen, Goose, Copilot) launched as child processes with injected env; ACP-first dispatch with bespoke fallback.
    43Ôćĺ  - Context bus (`AutumnBus`) = localhost HTTP server with `/mcp` (StreamableHTTP JSON-RPC) + `/hook/*` (session/stop/notify/pre-prompt/message-peer/task). Agents ride the bus via a zero-dep hook script.
    44Ôćĺ  - Canvas: 7 node types (screen, chat, sticky, analytics, youtube, terminal, remotion), 2 edge types (bus, navigation).
    45Ôćĺ  - "Commander" LLM prompt turns a user command into `DO_ACTIONS` plan (30 actions, verbatim recovered in `14-renderer-canvas.md`).
    46Ôćĺ  - Substitution table: October Supabase Ôćĺ own Supabase; Liveblocks Ôćĺ Yjs; FluidAudio Ôćĺ sherpa-onnx; Deepgram Ôćĺ drop; PostHog Ôćĺ drop; `october://` Ôćĺ `autumn://`.
    47Ôćĺ- **Sandbox constraint:** The actual development environment is the existing Next.js 16 project at `/home/z/my-project/`. Since Electron cannot run in this sandbox the way the original spec describes, the realistic deliverable here is a **Next.js web app** that demonstrates the Autumn concept ÔÇö a spatial canvas + Commander + agent orchestration UI ÔÇö using the open web stack we have (Next.js + Prisma + z-ai-web-dev-sdk for the LLM/intent layer). The deep RE notes remain the source of truth for behaviour and data model.
